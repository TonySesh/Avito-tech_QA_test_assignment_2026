# Тест-кейсы для проверки API Avito
Host - https://qa-internship.avito.com

Ручки:
- Создать объявления
- Получить объявления по его идентификатору
- Получить все объявления по идентификатору продавца
- Получить статистику по айтем id

Особенности:
- Каждое созданное объявление получает уникальный идентификатор.
- Поля name, price, sellerId, statistics могут повторяться у разных объявлений.
- sellerId — целое число. Рекомендуем использовать значения в диапазоне 111111–999999, 
чтобы избежать пересечений с другими тестировщиками.

# 1. POST — создание объявления

# Тест-кейсы для POST /api/1/item

## Позитивные сценарии

### TC-POST-01: Успешное создание объявления
- **ID:** TC-POST-01
- **Название:** Создание объявления с валидными данными
- **Шаги:**
Отправить POST-запрос на /api/1/item
В теле запроса передать JSON со всеми обязательными полями
- **Тестовые данные:**
```json
{
  "sellerID": %d,
  "name": "Itachi",
  "price": 122221,
  "statistics": {
    "likes": 1,
    "viewCount": 11,
    "contacts": 1111
  }
}
```
- **Ожидаемый результат:** HTTP-статус 200
